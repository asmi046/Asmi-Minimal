# Техническое задания на ветстку

Необходимо сверстать несколько секций, по предоставленному техническому заданию.

## Основные положения

- При верстке используй все современные технологии но только те которые имеют повсеместную поддержку в разных браузерах
- Для стилизации scss
- Стили для каждой секции должны лежать в отдельном файле. Эти файлы должны лежать в директории public\scss\, начинаться с символа '\_' и подключаться в файле public\scss\main.scss
- В начале каждого файла со стилями должно быть подключение файла с переменными, реализованное следующим образом: @use 'vars' as \*;.
- Если в ходе выполнения задания понвдобится вынести часть повторяющихся стилей то сделай отдельный файл и подключи его так чтобы можно было использовать в других фалах стилей.
- При создании стилей используй переменные которые описанны в файле public\scss_vars.scss. Не используй другие имена переменных! Используй только те которые есть в указанном файле.
- Каждая секция должна быть обернута в семантический тег section с осмысленным классом и Id. Внутри каждой секции должен быть контейнер описанный следующим образом <div class="container"> </div>. Всю разметку секции делай внутри данного контейнера.
- Без прямой необходимости или прямых указаний описанных в этом ТЗ не стилизуй секции и вложенный контейнер, стили для них уже прописанны.
- Без указаний в тексте задания не делай эффектов наведения для элементов.
- Если по ходу выполнения работ нужно создать кнопку по умолчанию делай ее при помощи тега a с прописанным классом button. Стили для кнопки уже прописанны.
- Созданные в ходе работы секции помещай в файл resources\views\index.blade.php перед закрывающим тегом @endsection

## Адаптация под устройства (общие требования)

- все секции должны быть адапрированы для мобильных устройств
- при написании стилей адаптации стариайся их минимизировать и не преписывать лишнее
- Мобильное устройство начинается от ширины экрана 520px
- Планшеты начинается от ширины экрана 820px
- Прими во внимание что для слайдеров у которых количество отображаемых элементов на десктоп версии > 1 нужно прописать брекпоинты при инициализации.
- Ниже в задании для каждой секции могут быть описанны требования для ее мобильной адаптации, учитывай эти требования.

Далее буду отправлять задания на верстку секций.

## Секция шапка сайта

В секции распологаются следующие элементы

- логотип обернутый в ссылку на главную файл public/img/logo-black.svg
- секция с контактами в которой:
    - меню вставь компонентом <menues.punct></menues.punct>
    - E-mail вставь ссылкой адрес почты возьми из конфига contacts.email
    - ссылки на мессенджеры, код для вставки возьми ниже.
    - телефон вставь не изменяя <a class="phone" href="tel:+7{{ phone_format(config('contacts.phone')) }}">{{ config('contacts.phone') }}</a>

### Ссылки на мессенджеры

```html
<a href="{{ config('contacts.tg_link') }}" class="sl_link tg_link">
    <svg class="sprite_icon __black">
        <use xlink:href="#tg_icon_black"></use>
    </svg>
</a>

<a href="{{ config('contacts.ws_link') }}" class="sl_link ws_link">
    <svg class="sprite_icon __black">
        <use xlink:href="#ws_icon_black"></use>
    </svg>
</a>
```

Все элементы расположенны в одну линию.

На мобильном устройстве остается только телефон и ссылки на соцсети

Разметку размести в файле public/scss/\_header.scss в этомфайле можно все удалить.
